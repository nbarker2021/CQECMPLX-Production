# Lib Kernel Deployment Proof

## Identity

`Lib Kernel` is the above-kernel library memory for CQE/CMPLX. It maps reusable
methods, rules, proofs, and lib-forge surfaces so the sidecar can pull behavior
by identity instead of carrying ad hoc internal routes.

## Adapter Boundary

The Lib Kernel is read as forms and reusable proof routes. Any move from a
source library into production must identify the reusable behavior, then route
it through CQE and the Universal Adapter. Binary movement is not expected for
this layer, but remains governed by the Binary Boundary Adapter.

## Production Split

Reusable behavior can promote into the production lib. Product-specific notes,
workspace-only scaffolds, and corpus material remain staged until a proof doc
names an exact slice.

## State

`slice-ready`

The first safe slice is the `lib_kernel` form/index structure itself. Individual
library implementations still need source-by-source proof review before import.
