## 🔄 Contribution Process

1. **Fork the repository** (if external)
2. **Create a feature branch**: `git checkout -b feature/my-feature`
3. **Add license headers** to all new files
4. **Ensure REUSE compliance**: Run `reuse lint` (if available)
5. **Commit with sign-off**: `git commit -s -m "feat: my feature"`
6. **Submit pull request** with full description

## ⚠️ Restrictions

**Your contribution CANNOT:**
- Include code from proprietary sources
- Violate any patents
- Be intended for commercial use
- Remove or alter license terms

## 📋 Commit Message Format

```
type(scope): short description

Longer description if needed

Signed-off-by: Your Name <email@example.com>
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

## 🧪 Testing

- All tests must pass
- New features require tests
- Maintain research-only compliance
