# CMPLX-MORSR CQE Bridge

CMPLX-MORSR uses CQE to connect runtime diagnostic behavior to formal structure, product use, and library ownership.

```text
MORSR pulse/recenter engine
  -> E8/C-centroid formal structure
  -> diagnostic product surface
  -> cmplx.morsr / cmplx.receipt / cmplx.nsl / cmplx.morphon bindings
```

## Default

Everything generated by CMPLX-MORSR is library-bound unless CQE marks it as a new datum.

## CQE Responsibilities

- Convert runtime context into a C centroid.
- Classify pulse and recenter events as engine behavior.
- Bind 240-direction pulse structure to E8 formal math.
- Bind confirmed readings to morphon-compatible pode/node records.
- Bind conservation to NSL.
- Bind trace persistence to receipts.
- Require hidden-result honesty for non-math diagnostics.
