# CMPLX-MORSR Library Bindings

These are the existing workspace libraries CMPLX-MORSR should use.

## `cmplx.morphon`

Use for:

- C centroid identity records
- pode/node records
- parent/child links after recentering
- payload state snapshots

## `cmplx.nsl`

Use for:

- pulse/recenter conservation score
- gate mode selection
- delta phi receipt field
- rejection/amortization/signal reason

## `cmplx.receipt`

Use for:

- pulse receipts
- hidden guess receipts
- reveal receipts
- score receipts
- recenter receipts
- contamination receipts

## `cmplx.morsr`

Use only as a temporary diagnostic port shape until the later supplied MORSR arrives.

## `lattice_forge`

Use for:

- Forge-family inheritance pattern
- honesty/open-obligation discipline
- umbrella packaging convention
- proof/diagnostic boundary language

## Market/Wave Engines

Use as signal sniffers for:

- centroid tension
- phase shifts
- wave mismatch
- success and failure point localization

Do not treat these as finance-only tools.
