# Analog Forge Replacements And Variables

## Purpose

This guide lets an operator run the Analog Forge kit even when the exact
suggested tool is unavailable. The replacement rule is:

```text
preserve function first
preserve color second
preserve physical form third
```

Do not stop a run because a perfect material is missing. Replace the tool with
something that performs the same state role and receipt the substitution.

## Replacement Receipt

Every substitution should be recorded:

```text
original suggested tool
replacement tool
preserved function
lost function, if any
color mapping
boundary effect
operator initials
receipt id
```

## Color Replacement

If exact colors are missing, preserve color role with labels.

| Role | Preferred | Replacement |
|---|---|---|
| red | red material | label `RED`, triangle mark, R symbol |
| green | green material | label `GREEN`, square mark, G symbol |
| blue | blue material | label `BLUE`, circle mark, B symbol |
| white | white material | label `PROOF`, open circle, W symbol |
| black | black material | label `OBLIGATION`, filled square, K symbol |
| grey | grey material | label `UNRESOLVED`, dotted border |
| clear | clear material | label `OVERLAY`, transparent or dashed border |
| neon | neon material | label `BOUNDARY`, starburst mark |

## If You Lack Eight Copies

Eight copies per color is the ideal. If you do not have all eight, use slot
labels:

```text
slot 1
slot 2
slot 3
slot 4
slot 5
slot 6
slot 7
slot 8
```

The important thing is not eight physical luxury objects. The important thing is
that the operator does not collapse eight possible local readout positions into
one unmarked object.

## If You Lack A Physical Tool Entirely

Use a drawn version.

```text
drawn token = circle with ID
drawn string = line with source/target
drawn sticker = boxed label
drawn dice = recorded branch sampler
drawn card = rank/suit written on paper
drawn balsa = edge segment with endpoints
```

Drawn tools are valid if the receipt states they are drawn replacements.

## Replacement Safety Rule

If a replacement changes the boundary, mark it in neon and receipt it.

Examples:

```text
rubber band instead of string -> elasticity added
marker line instead of string -> relation no longer movable
paper token instead of magnet -> token may drift
phone photo instead of receipt scan -> image quality obligation
```

The replacement is still valid, but its changed behavior becomes part of the
run.

## Completion Rule

A replacement is valid when:

```text
its function is named
its color role is named
its lost behavior is named
its receipt exists
```

Without those four records, the replacement remains informal.
